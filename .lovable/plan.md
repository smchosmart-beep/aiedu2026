## 목표
AI 처방전(수업 흐름 / 평가 포인트 / 흔한 함정 / 현장 스크립트)의 **가독성**을 높여, 컨설턴트가 현장에서 빠르게 스캔할 수 있도록 정보 위계와 타이포그래피를 개선.

## 현재 문제점
- `flow`(수업 흐름): 한 줄에 "도입: ...", "탐구: ...", "평가: ..." 같은 긴 문장이 단순 `<ol>` 번호 목록으로 렌더링됨 → 단계 라벨과 본문이 시각적으로 분리되지 않음. 한 문단처럼 보여 흐름이 안 잡힘.
- `evaluationPoints` / `commonTraps`: 같은 크기·같은 색의 본문 텍스트 + 작은 아이콘. 두 섹션이 시각적으로 거의 동일해서 구분이 약함.
- `consultingScript`: 4문장 한 덩어리로 출력. 따옴표 블록이 길어 한 호흡으로 읽기 어려움.
- 전체 위젯이 세로로 길어 스크롤 피로도가 높음. 섹션 간 여백·구분이 약함.

## 개선 범위
오직 `src/components/consult/Dashboard.tsx`의 "🤖 AI 맞춤 처방전" 위젯 내부 렌더링만 수정. 데이터 스키마, 서버 함수, 다른 위젯은 변경하지 않음.

## 변경 사항

### 1. 수업 흐름 — 단계 카드 형태로 분해
현재: 단일 `<ol>` + `<li>` 텍스트.
개선:
- 각 단계 문자열을 `"라벨: 본문"` 패턴으로 파싱 (`split(/[:：]/, 2)`).
- 라벨 파싱 실패 시 단계 번호("1단계","2단계"...)로 폴백.
- 각 단계를 **번호 배지 + 라벨(굵게) + 본문(부드러운 회색)** 3단 레이아웃 카드로 출력. 카드 사이 세로 구분선 또는 좌측 타임라인 라인.
- 모바일 우선: 좌측 number circle (primary 배경), 우측 텍스트 컬럼.

### 2. 평가 포인트 — 체크리스트 박스
- `bg-emerald-50/40` 톤의 부드러운 배경 박스 + 좌측 강조 보더(green).
- 항목마다 둥근 체크 아이콘 + 본문. 항목 간 1px 점선 separator.
- 섹션 타이틀에 "교과 + 윤리 융합" 같은 부제 위치 정리.

### 3. 흔한 함정 — 경고 톤 박스
- `bg-amber-50/50` + 좌측 amber 보더로 평가 포인트와 색으로 즉시 구별.
- 각 항목 앞에 "⚠" 대신 둥근 amber 배지(번호) — "함정 1", "함정 2" 식 라벨.

### 4. 현장 스크립트 — 문장 단위 줄바꿈 + 인용 디자인 강화
- `consultingScript`를 `/(?<=[.?!。])\s+/`로 문장 단위 분리 후 각 문장을 `<p>`로 렌더 (문단 간 여백).
- 큰 따옴표 글리프(") 장식 추가, 좌측 강조 바 두껍게(border-l-4 → border-l-[6px]), 배경을 `bg-primary/5` → `bg-gradient-to-br from-primary/8 to-primary/3` 로 입체감.
- 하단에 "현장 화법" 라벨 배지.

### 5. 공통 타이포그래피·간격 정비
- 본문 가독성: 현재 `text-[15px] leading-relaxed` 유지하되, 평가/함정 항목은 `leading-7`로 행간 확대.
- Section 타이틀: 크기 살짝 키우고(`text-sm font-semibold` → `text-[15px] font-bold`) 아이콘과 간격 통일.
- 섹션 사이 `space-y-5` → `space-y-7`로 호흡 추가.

### 6. (선택) 모델 정의 강조
- `modelDefinition` 카드를 `bg-muted/40` rounded-2xl로 감싸 본문에서 시각적으로 분리.

## 변경하지 않는 것
- 서버 함수 응답 스키마, 프롬프트, 캐싱, 재생성 동작
- "🎯 핵심 수업·평가 모델 처방 (기준 모델)" 등 다른 위젯
- 색 토큰: 새 색은 추가하지 않고 Tailwind 의미 토큰 + 기존 `primary`, `amber`, `emerald` 활용

## 위험 / 예외
- `flow` 항목이 라벨:본문 패턴이 아닌 경우 → 폴백(번호만 표기)로 안전하게 처리.
- `consultingScript`가 한 문장일 경우 → split 결과 1개여도 정상 렌더.
