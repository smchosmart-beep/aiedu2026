## 방향 전환

이전 계획의 "도구 사실 시트(factsheet)" 접근을 폐기합니다. 대신 **AI가 구체적인 도구 명칭을 일절 언급하지 않도록** 프롬프트에서 강제합니다. 모르는 제품을 어설프게 끌어오는 것보다, 카테고리 단위로 일반화해 진술하는 편이 현장 신뢰도와 재사용성 모두 더 높습니다.

## 변경 내용

### `src/lib/prescription.functions.ts`

**1. 시스템 프롬프트 — 도구 명칭 금지 룰로 교체**

기존의 "선호 도구의 ○○기능을 정확히 지칭하라" 강제 규정을 제거하고, 다음으로 대체:

- **고유명사 금지**: 처방 본문(`title`, `modelDefinition`, `flow`, `evaluationPoints`, `commonTraps`, `consultingScript`)에 특정 에듀테크 제품명·서비스명을 절대 쓰지 않는다. (예: "옥수수", "똑똑수학탐험대", "칸아카데미", "뤼튼", "캔바", "패들렛", "ChatGPT" 등)
- **일반화 표기 사용**: 대신 카테고리/기능 명칭으로만 기술한다.
  - 예: "수학 코스웨어", "AI 진단 리포트 도구", "생성형 챗봇", "{과목}과 에듀테크", "협업 화이트보드", "디지털 교과서"
- **함수적 지칭 권장**: "○○ 기능이 있는 도구를 활용해" 형태로 능력을 기술하고, 어떤 제품을 쓸지는 교사 판단에 맡긴다.
- **단원 예시 가드레일**(유지): 가짜 단원명·성취기준 코드는 만들지 않고 "학년 수준에 맞는 단원" 같은 일반화 표현을 사용한다.

**2. 입력 활용 방식 변경**

`buildUserPrompt`에서 `선호 에듀테크 도구` 줄은 그대로 모델에게 전달하되, 다음 한 줄을 명시적으로 추가:

> "위 '선호 도구' 목록은 교사 입력 그대로의 참고치다. 처방 본문에서는 도구를 카테고리({과목}과 에듀테크, 수학 코스웨어 등)로만 지칭하고 제품명을 절대 노출하지 마라."

**3. 사후 검증(가벼운 가드)**

서버 함수 응답 직전에 정규식 기반의 간단한 sanitizer를 추가:

- 잘 알려진 한국 에듀테크 제품명 블랙리스트(약 15~20개)에 매칭되면 콘솔에 경고를 남기고, 매칭된 토큰만 일반 카테고리 단어("에듀테크 도구")로 치환.
- 재호출은 하지 않음 (비용·지연 절감). 1차 방어선으로만 동작.

### UI

`src/components/consult/Dashboard.tsx`는 변경 없음. 기존 가독성 레이아웃 유지.
