## 문제
설문(StepShell) 진입 후 진행 헤더에는 ‹ 뒤로(이전 단계) 버튼만 있고, 홈으로 빠져나갈 방법이 없음. 사용자가 중간 이탈 시 막힘.

## 해결
StepShell 상단 헤더 우측에 “홈” 아이콘 버튼을 추가해 언제든 랜딩 페이지(`/`)로 돌아갈 수 있게 함. 첫 단계가 아닌 경우 입력값이 사라지므로, 클릭 시 confirm 다이얼로그로 한번 확인 후 이동.

### 변경 파일
- `src/components/survey/StepShell.tsx`
  - 헤더 우측 "단계 표시" 옆에 홈 아이콘 버튼 추가 (`Home` from lucide-react)
  - `useNavigate()`로 `/` 이동
  - step > 0 이고 마지막 단계 미완료 상태일 때 `window.confirm("진단을 중단하고 홈으로 돌아갈까요? 입력한 내용은 저장되지 않아요.")` 확인

### UX
- 홈 버튼은 작은 ghost 아이콘 버튼, 진행률 바와 단계 카운터 사이/오른쪽 끝에 배치
- 모바일 384px 뷰포트에서도 헤더 레이아웃 깨지지 않도록 너비 조정
