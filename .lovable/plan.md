## 1. 위젯 구성 변경

기존 4개 위젯 → 다음 3개 위젯으로 단순화:

1. **🎯 핵심 수업·평가 모델 처방** (메인, 대폭 확장)
2. **⚠️ 현재 겪고 있는 어려움 (확인용)** — 선생님이 체크한 difficulties + 계정 상태를 라벨/뱃지로만 보여주는 요약. 처방 문구 없음.
3. **💬 현장 스크립트** (유지)

제거: `💻 기기·인프라 맞춤 처방`, `📝 행정·계정 맞춤 처방` 위젯과 관련 `infraItems` / `adminItems` 로직.

## 2. 핵심 처방 위젯 확장 — TYPE_META 재설계

각 유형 항목을 다음 구조로 확장:

```ts
{
  label, stage,
  oneLiner: string,            // 한 줄 요약 (현 prescription 대체)
  coreModel: string,           // 핵심 수업·평가 모델 이름 + 정의 (2~3문장)
  lessonFlow: string[],        // 1차시 운영 흐름 4~6단계 (도입→전개→평가→피드백)
  evalFocus: string[],         // 평가 포인트 3~4개 (무엇을 보는가)
  pitfalls: string[],          // 주의/함정 2~3개
  nextStep: string,            // 다음 단계로 가는 트리거 1문장
  script: string,              // 현장 스크립트 (유지)
}
```

### 유형별 콘텐츠 (요약 — 실제 본문은 더 구체적으로 작성)

- **A · 입문형** — 모델: "AI 코스웨어 채점 자동화 + 오답 군집 진단". 흐름: 진단지 PIN 접속 → 학생 풀이 → 자동 채점 → 오답 유형 군집 확인 → 보충 그룹 재편성. 평가: 정답률, 오답 패턴, 풀이 시간. 함정: 채점만 자동화하고 피드백은 교사가 일괄 구두로 끝내버리는 것. 다음 단계: 확보된 10분을 학생 설명 활동으로 전환.
- **A-B · 입문→표준 전환형** — 모델: "AI 생성 문항 + 학생 풀이 설명 1차시 실험". 흐름: 교사가 AI에게 단원 핵심 문제 3개 생성 의뢰 → 학생 풀이 → 학생이 AI에게 자기 풀이를 설명 → 설명 로그 캡처 → 교사 1줄 코멘트. 평가: 풀이 정확성 + 설명의 논리성. 함정: AI 문제의 난도 검수 누락. 다음 단계: 같은 모델을 단원 전체로 확장.
- **B · 표준형** — 모델: "AI 초안 비판적 대화(Critical Dialogue) 평가". 흐름: 발문 제시 → 학생이 AI 초안 받기 → 초안의 오류·근거 부족 지점 표시 → 학생-AI 반박 대화 → 최종 정정안 제출. 평가: 초안 대비 수정 횟수, 근거 인용, 반박 논리. 함정: 학생이 초안을 그대로 제출, 대화 로그 미저장. 다음 단계: 한 단원에 한해 학생이 AI를 수학적으로 반박하는 활동 도입.
- **B-C · 표준→전문가 전환형** — 모델: "AI 응답 반례(Counterexample) 미니 프로젝트". 흐름: 단원 1개 선정 → AI에 일반화 질문 → 학생 모둠이 반례 후보 탐색 → 반례 검증/증명 → 동료 평가. 평가: 반례의 수학적 타당성, 증명 단계, 동료 피드백 수용도. 함정: 단순 "AI가 틀렸다"로 끝나고 수학적 근거 부재. 다음 단계: 학기 단위 인간 주체성 프로젝트로 확장.
- **C · 전문가형** — 모델: "AI 알고리즘 검증 기반 수학적 주체성 수업 (예: 7의 배수 판별 알고리즘 비판)". 흐름: AI에게 판별법 코드/규칙 의뢰 → 학생이 알고리즘 분해 → 반례·예외 사례 설계 → 수학적 원리(합동식 등)로 한계 증명 → 개선안 발표. 평가: 알고리즘 이해, 수학적 증명, 대안 설계, 메타 성찰. 함정: 기술 비판이 감상 수준에 머무름. 다음 단계: 학교 단위 사례 공유.

## 3. UI 렌더링

핵심 처방 위젯 내부:
- 상단: 유형 뱃지 + 점수 + 5단계 진행 바 (기존 유지)
- `stage` + `oneLiner`
- 섹션 1 "🧩 모델 정의" — `coreModel`
- 섹션 2 "🗺️ 1차시 운영 흐름" — `lessonFlow` 번호 리스트
- 섹션 3 "📊 평가 포인트" — `evalFocus` 체크 리스트
- 섹션 4 "⚠️ 흔한 함정" — `pitfalls` 리스트
- 섹션 5 "➡️ 다음 단계" — `nextStep` 강조 박스

각 섹션은 작은 제목 + 리스트로 구성, 카드 안에서 `space-y-4`로 구분.

## 4. 어려움 확인 위젯

- 표시 항목: `data.difficulties` 라벨 뱃지들 + `data.account === "none"` 또는 `"shared"`일 때 계정 상태 뱃지
- 처방/조언 문구 없음. "선생님이 체크하신 항목입니다"라는 안내 한 줄만.
- 선택된 항목이 없으면 "특이 사항 없음" 표시.
