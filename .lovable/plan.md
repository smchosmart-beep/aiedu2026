## 목표
A/B/C 3단계 고정 분류를 점수 기반 5단계(A, A-B, B, B-C, C)로 확장하여 경계 상황의 교사에게 더 정밀한 처방 제공.

## 1. 점수 산정 방식

세 가지 신호를 0~1 점수로 환산하여 합산 → **0(완전 A) ~ 4(완전 C)** 스펙트럼 점수 산출.

**신호 1: 계정 환경 (가중치 1.0)**
- `none` → 0
- `shared` → 1
- `personal` → 2

**신호 2: 교사 숙련도 (가중치 1.0)**
숙련도 다중 선택의 평균값 사용 (1~4) → 0~3으로 정규화
- 평균 1 → 0
- 평균 2 → 1
- 평균 3 → 2
- 평균 4 → 3

**신호 3: 수업 설계 갈증 (가중치 보정)**
- `difficulties`에 `design` 포함 → +1 보정 (전문가 방향 가중)
- 포함하지 않고 `infra`/`account`만 있음 → −0.5 (입문 방향)

**최종 점수 = 계정점수 + 숙련도점수 + 보정**
범위: 약 −0.5 ~ 6.0 → 5구간으로 분할

| 점수 구간 | 유형 |
|---|---|
| ≤ 1.0 | A (입문형) |
| 1.0 < x ≤ 2.0 | A-B (입문→표준 전환) |
| 2.0 < x ≤ 3.5 | B (표준형) |
| 3.5 < x ≤ 4.5 | B-C (표준→전문가 전환) |
| > 4.5 | C (전문가형) |

> 임계값은 시연 데이터 분포 보고 1차 조정 가능. 하드 오버라이드: `account === "none"`이면 무조건 A (계정 없으면 어떤 모델도 못 굴림).

## 3. 처방전 화면 변경

핵심 처방 위젯에서 단일 유형 표시 → **주 유형 + 인접 유형 보조 표시**:
- 5단계 진행 바 (A · A-B · B · B-C · C) 위에 현재 위치 점 표시
- 점수 수치도 작게 노출 (예: `점수 2.7 / 5단계 중 B`)
- 기존 처방/스크립트는 매핑된 유형의 것을 그대로 사용

## 4. 변경 파일

- `src/lib/classify.ts` — 점수 기반 로직으로 재작성
- `src/components/consult/Dashboard.tsx` — `typeMeta`에 A-B, B-C 항목 추가, 5단계 진행 바 UI 추가, 점수/주유형 표시
- (영향 확인) `src/lib/storage.ts`에서 `ConsultType`을 직접 분기하는 코드가 있는지 점검 후 필요 시 수정

## 5. 미적용 사항
- 가중치/임계값은 첫 구현 후 실데이터로 튜닝 (지금은 위 기본값으로 시작)
- 다중 후보 표시(B 60% / C 40% 그래프 방식)는 채택하지 않음
