# AI 디지털 활용 선도학교 수업 평가 컨설팅 사전진단 도구

디지털 선도학교 현장의 디지털 도구 활용 실태를 파악하고, 동료 교사·연구자가 **컨설팅 기록**을 남겨 함께 성장할 수 있도록 돕는 웹 애플리케이션입니다.

> 배경: 디지털 선도학교 현장의 기기·계정·평가 환경을 구조화해 수집하고 누적된 사례를 토대로 컨설팅을 제공하기 위한 도구로 개발되었습니다.

---

## 이런 분께 도움이 됩니다

- **디지털 선도학교 담당 교사** — 우리 학교의 디지털 수업 환경을 한 번에 정리하고, 다른 학교 사례와 비교해 보고 싶은 분
- **수업 평가 컨설턴트·장학사** — 사전 진단 결과를 바탕으로 맞춤형 컨설팅 의견을 남기고 싶은 분
- **AI·디지털 교육 연구자** — 학교 현장의 디지털 활용 실태와 어려움을 구조화된 형태로 살펴보고 싶은 분

별도 가입이나 로그인 없이 누구나 바로 사용할 수 있습니다.

---

## 무엇을 할 수 있나요?

### 1. 우리 학교 디지털 환경 진단하기
지역·학교, 학생들이 쓰는 기기와 계정, 교사가 느끼는 어려움, 선호하는 도구, 평가 목표 등을 단계별 설문으로 정리합니다. 제출하면 **6자리 코드**(예: `ABCDE2`)가 자동으로 발급되어 언제든 다시 열어볼 수 있습니다.

### 2. 다른 학교 사례 찾아보기
- **코드로 바로 열기** — 발급받은 6자리 코드로 본인 또는 동료의 진단 결과를 엽니다.
- **학교급·지역으로 찾기** — 초·중·고, 지역, 학교명 등으로 비슷한 환경의 학교를 둘러봅니다.
- **키워드로 검색** — "패들렛", "AI 튜터", "평가 루브릭"처럼 관심 있는 주제로 사례를 찾습니다.

검색 결과 카드는 **컨설팅 기록이 많이 쌓인 사례일수록 진한 네이비 색**으로 표시되어, 활발하게 논의된 사례를 한눈에 알 수 있습니다.

### 3. 컨설팅 의견 주고받기
누구나 컨설턴트 이름·내용·참고 링크와 함께 **4자리 PIN**을 설정해 의견을 남길 수 있습니다.
- 본인이 설정한 PIN을 알아야만 **수정·삭제**가 가능합니다.
- 다른 사람의 PIN은 어디에서도 노출되지 않습니다.

## 사용 방법 (3단계)

1. **설문 응답** — 첫 화면에서 "시작하기"를 눌러 설문을 순서대로 작성합니다. 중간에 멈추지 말고 끝까지 진행해 주세요.
2. **6자리 코드 저장** — 제출이 끝나면 발급된 코드가 화면에 나타납니다. **꼭 따로 메모하거나 캡처해 두세요.** 이 코드가 결과에 다시 접근하는 유일한 열쇠입니다.
3. **결과·컨설팅 활용** — 상단의 "결과 보기"에서 코드를 입력하거나 학교급·키워드로 다른 사례를 찾아보고, 필요하면 컨설팅 의견을 남기거나 받습니다.

## 자주 묻는 질문 (FAQ)

**Q. 6자리 코드를 잃어버렸어요.**
안타깝지만 코드 없이는 본인 응답을 직접 찾기 어렵습니다. 학교명·지역 등으로 검색해 본인 응답을 찾거나, 새로 설문을 다시 작성해 주세요.

**Q. 컨설팅에 쓴 4자리 PIN을 잊었어요.**
보안을 위해 PIN은 해시 처리되어 저장되며 복구할 수 없습니다. 새 의견으로 다시 작성해 주세요.

**Q. 제 응답이 누구에게 공개되나요?**
이름과 학교 등 입력하신 정보는 검색 결과에 표시될 수 있습니다. 민감한 개인정보(전화번호, 학생 이름 등)는 입력하지 말아 주세요.

**Q. 누가 컨설팅 의견을 남길 수 있나요?**
별도 가입 없이 누구나 가능합니다. 신뢰할 수 있는 의견을 위해 컨설턴트 이름과 소속을 정확히 적어 주시기를 권장합니다.

**Q. 모바일에서도 잘 되나요?**
네, 휴대폰·태블릿·PC 모두 지원합니다.

## 개인정보·보안 안내

- 입력하신 응답은 **컨설팅과 교수-학습 모델 연구 목적**으로만 활용됩니다.
- 데이터베이스는 외부에서 직접 접근할 수 없도록 잠겨 있으며, 검증된 서버 경로를 통해서만 읽고 쓸 수 있습니다.
- 컨설팅 PIN은 원문이 아닌 **암호화된 형태**로 저장되어, 운영자도 원래 값을 알 수 없습니다.

### 보안 모델

- `surveys`, `consultations` 테이블은 **RLS로 완전히 잠금**. anon/authenticated 직접 접근 불가.
- 모든 읽기·쓰기는 `service_role`을 사용하는 **서버 함수**(`src/lib/*.functions.ts`)를 통해서만 가능.
- 클라이언트로는 `pin_hash`, `pin_salt`를 노출하지 않으며, 대신 `canModify` 플래그만 전달.
- 컨설팅 PIN은 salt + SHA-256 해시 + timing-safe 비교로 검증.
